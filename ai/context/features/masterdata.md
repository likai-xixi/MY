# Masterdata Focused Context

Feature: `masterdata`

## Current Runtime

- R-10B owns the nine approved masterdata resources: product category, product series, product model, material category, material item, accessory category, accessory item, sales option category, and sales option value.
- R-10D changes create behavior so codes are generated by the backend, not entered by users.
- Code format is `prefix + yyyyMM + 6 digit monthly sequence`.
- Prefixes are `PC`, `PS`, `PM`, `MC`, `MI`, `AC`, `AI`, `SOC`, and `SOV` in the resource order above.
- The shared business generator lives at `ruoyi-business/src/main/java/com/ruoyi/business/common/code/BusinessMonthlyCodeGenerator.java`; it must not move to `ruoyi-common`.

## Boundaries

- Do not create sales-order runtime.
- Do not create field-scheme, formula, or technical-decomposition runtime.
- Do not create inventory, BOM, production, scan, drawing, shipment, finance, or receipt runtime.
- Do not modify customer runtime or idempotency runtime in masterdata code-generation work.
- `beforeSalesOrder` remains blocked.

## Verification Focus

- `tests/masterdata-runtime.test.js` guards the nine resources, prefix mapping, code format, add-without-code behavior, edit code immutability, uniqueness, bounded retry, frontend add/edit code behavior, and forbidden runtime absence.
