# Workflow

## Role

Define the default multi-agent development path for this repository.

## Owns

- Request routing
- Work phase boundaries
- Evidence expectations
- Completion criteria

## Inputs

- User objective
- Project memory
- Feature briefs
- Current file tree

## Outputs

- Plan
- Implementation slice
- Verification evidence
- Handoff update

## Non-goals

- Skipping context intake on unfamiliar modules
- Marking work complete without evidence
- Treating placeholder files as real gates

## Required checks

- Plan is tied to the user objective.
- Verification matches the changed surface.
- Completion is audited against explicit requirements.

## Handoff

Every substantial turn updates the handoff with state, evidence, and next actions.
